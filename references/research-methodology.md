# Research Methodology Reference

This document outlines the research methodology principles that underpin the deep search skill.

## Problem Decomposition

Effective research starts with breaking a complex question into manageable sub-questions:

- **MECE Principle**: Sub-questions should be Mutually Exclusive, Collectively Exhaustive
- **Dimensional Analysis**: Consider the topic from multiple dimensions:
  - Technical / Implementation
  - Economic / Market
  - Social / User impact
  - Temporal (past, present, future)
  - Comparative (alternatives, competitors)
- **Assumption Surfacing**: Identify and list implicit assumptions in the original question

## Query Optimization

Search queries should be crafted strategically:

- **Specificity Ladder**: Start broad, then narrow based on initial results
- **Synonym Variation**: Use different terms for the same concept across queries
- **Boolean Logic**: Combine terms to filter noise (e.g., "RISC-V laptop -Arduino")
- **Source Targeting**: Add site-specific terms when authoritative sources are known
- **Temporal Filtering**: Include year or date ranges for time-sensitive topics
- **Language Switching**: Search in the language most likely to have expert content

## Source Evaluation (CRAAP Test)

Evaluate each source against these criteria:

| Criterion | Questions to Ask |
|-----------|-----------------|
| **Currency** | When was it published? Is the information current for the topic? |
| **Relevance** | Does it directly address the research question? |
| **Authority** | Who is the author? What are their credentials? |
| **Accuracy** | Can claims be verified? Are sources cited? |
| **Purpose** | Is the content informational, commercial, or opinion? |

### Source Hierarchy (strongest to weakest)

1. Peer-reviewed research / official documentation
2. Government / institutional reports
3. Industry analysis from established firms
4. Quality journalism (named reporters, fact-checked outlets)
5. Expert blog posts / conference talks
6. Community forums / social media (use for signals, not facts)

## Synthesis Techniques

### Triangulation

- Verify key claims across 3+ independent sources
- Note when only a single source supports a claim
- Flag contradictions explicitly rather than silently choosing one version

### Chronological Threading

- Trace how information has evolved over time
- Note when "facts" from older sources have been superseded
- Identify emerging trends vs established consensus

### Confidence Levels

Apply confidence ratings to findings:

| Level | Meaning | Source Requirement |
|-------|---------|-------------------|
| **High** | Well-established, widely confirmed | 3+ independent authoritative sources |
| **Medium** | Likely accurate, some uncertainty | 1-2 authoritative sources |
| **Low** | Preliminary, unconfirmed | Single source or expert opinion only |
| **Speculative** | Reasonable inference, no direct evidence | Analyst interpretation |

## Common Pitfalls

- **Confirmation Bias**: Searching only for evidence that supports a hypothesis
- **Recency Bias**: Overweighting the latest news vs established fundamentals
- **Authority Bias**: Accepting claims from "experts" without verification
- **Survivorship Bias**: Focusing on successful cases while ignoring failures
- **Translation Errors**: Misinterpreting content in a non-native language
